## Current Standards And Time-Sensitive Notes

- WCAG 2.2 remains the current W3C baseline, and the WCAG overview still encourages teams to use the latest version. That overview page is currently marked updated on 2026-05-26. Source: https://www.w3.org/WAI/standards-guidelines/wcag/
- Current W3C guidance for Success Criterion 2.4.11 Focus Not Obscured (Minimum) says a focused component must not be entirely hidden by author-created content, and it explicitly calls out sticky banners plus persistent user-opened content such as chat windows as common failure shapes. Passing patterns include modal treatment, scroll padding, displacement, or another reveal path that does not force the user to backtrack through the obscuring surface. Source: https://www.w3.org/WAI/WCAG22/Understanding/focus-not-obscured-minimum.html
- WCAG 2.2 target-size minimum guidance still centers on a `24 by 24 CSS pixel` minimum unless an exception applies. Current [ConsentManager.tsx](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/src/components/ConsentManager.tsx) still keeps the collapsed launcher above that floor, so this standard does not support reopening `TB-0032` on current head. Source: https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html
- Current WAI-ARIA APG dialog guidance still expects focus to move inside a dialog when it opens and strongly recommends a visible close button in the dialog tab sequence. That remains the relevant standard for the shared mobile sidebar sheet and any modal consent/settings flow. Source: https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/
