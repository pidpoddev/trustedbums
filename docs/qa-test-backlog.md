## Current Standards And Time-Sensitive Notes

- Playwright still recommends authenticating once in a setup project, storing `storageState`, and reusing that state across tests rather than signing in inside each spec. Source: [Playwright Authentication](https://playwright.dev/docs/auth).
- Playwright visual regression guidance still centers on `expect(page).toHaveScreenshot()` and matching baselines in the same environment to reduce rendering noise. Source: [Playwright Visual comparisons](https://playwright.dev/docs/test-snapshots).
- Playwright’s current accessibility guidance still points teams toward automated scans such as axe plus keyboard/semantic checks for critical flows. Source: [Playwright Accessibility testing](https://playwright.dev/docs/accessibility-testing).
- Vitest still recommends native V8 coverage as the low-friction path and documents accurate AST-based remapping in current releases. Source: [Vitest Coverage](https://vitest.dev/guide/coverage.html).
