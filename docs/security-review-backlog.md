Align `supabase/config.toml` with any live `verify_jwt` exceptions so future deploys do not silently roll back a live hardening improvement.
- Acceptance criteria: no Clerk-backed privileged Edge Function keeps the token-supplied issuer fallback; live and repo `verify_jwt` settings agree for each function; and tests prove missing-issuer and wrong-issuer tokens fail closed for admin-only, client-admin, Bum, self-service, and public-with-bearer classes.
