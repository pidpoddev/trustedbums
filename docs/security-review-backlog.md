### P1 - [TB-0056] Move admin scrum tracker audit actors to a server-owned path
- Evidence: The new admin-only route in [src/App.tsx](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/src/App.tsx) and [src/layouts/AdminLayout.tsx](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/src/layouts/AdminLayout.tsx) exposes the new tracker UI in [src/pages/admin/AdminScrumTracker.tsx](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/src/pages/admin/AdminScrumTracker.tsx). The backing table in [supabase/migrations/20260609025228_add_admin_scrum_items.sql](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/supabase/migrations/20260609025228_add_admin_scrum_items.sql) is admin-only by RLS, but the browser helpers in [src/lib/portalApi.ts](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/src/lib/portalApi.ts) still send `created_by` and `updated_by` from `currentUserId`, and the migration does not add a database-side trigger or check that pins those actor columns to a trusted auth subject. The current business-rule source of truth in [docs/business-access-rules.md](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/docs/business-access-rules.md) also has no explicit rule for admin scrum tracker data, actor ownership, or audit expectations.
- Why it matters: The tracker is intended to become a durable operations and closeout ledger with evidence links, commit references, and closure notes. If the actor columns are client-controlled, admin audit history is forgeable, and incident review or accountability on internal changes stops being trustworthy.
- Recommendation: Move scrum-item writes behind a server-owned actor source. The database or a server-owned function should derive `created_by` and `updated_by` from the trusted auth subject and ignore client-supplied actor IDs. Add an admin-scrum business access rule covering visibility, allowed mutations, sensitive fields, and actor-audit expectations before the tracker becomes a relied-on operational record.
- Acceptance criteria: Non-admin users are denied at the route and data layer; client-supplied actor IDs cannot spoof another profile or blank out audit history; [docs/business-access-rules.md](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/docs/business-access-rules.md) has an admin scrum tracker rule; and targeted tests prove audit actor integrity plus admin-only access.
