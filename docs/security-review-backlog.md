## Time-Sensitive Threat Notes

- GitHub advisory `GHSA-qjx8-664m-686j` for `js-cookie`, updated May 21, 2026, marks `<= 3.0.5` affected and `3.0.7` patched. [Source](https://github.com/advisories/GHSA-qjx8-664m-686j)
- GitHub advisory `GHSA-w5hq-g745-h8pq` for `uuid`, updated May 21, 2026, marks `< 11.1.1` and certain `12.x`/`13.x` ranges affected. Trusted Bums still resolves `uuid@8.3.2` transitively in the current lockfile, so keep that dependency path visible until the owning package is assessed or updated. [Source](https://github.com/advisories/GHSA-w5hq-g745-h8pq)
- GitHub advisory `GHSA-2j2x-hqr9-3h42`, updated June 3, 2026, affects `react-router` `< 6.30.4`, but GitHub explicitly says Declarative Mode with `<BrowserRouter>` is not impacted. Current app source still uses `<BrowserRouter>` in [src/App.tsx](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/src/App.tsx). [Source](https://github.com/advisories/GHSA-2j2x-hqr9-3h42)
- Current Supabase API guidance says functions are controlled by `EXECUTE` grants rather than RLS, recommends reviewing any `SECURITY DEFINER` functions carefully, and recommends an opt-in exposure model or dedicated API schema. The current live helper cleanup reflects this guidance. [Source](https://supabase.com/docs/guides/api/securing-your-api)
- Current Supabase RLS guidance warns against using end-user-modifiable metadata for authorization decisions and recommends keeping internal helper surfaces out of exposed paths when possible. [Source](https://supabase.com/docs/guides/database/postgres/row-level-security)
- Current Clerk docs still say `unsafeMetadata` can be updated from the frontend with `User.updateMetadata()`, so Trusted Bums must continue treating it as onboarding input only, never as authoritative authorization state. [Source](https://clerk.com/docs/guides/users/extending)
- Current Cloudflare Turnstile guidance still expects server-side validation plus checks on expected `action` and hostname. [Source](https://developers.cloudflare.com/turnstile/get-started/server-side-validation/)
