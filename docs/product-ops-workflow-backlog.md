- Admin handoffs: Bum target responses and client-to-Bum intro requests surface in `/admin/handoffs` -> Admin claims and advances status -> Product Ops still needs due-state and last-touch parity with contact submissions before those queues scale.

- Access-review and bootstrap exceptions: `profile-bootstrap` and `client-team` create `client_company_access_requests` when same-domain auto-approval is unsafe or when manual review is required -> Admin or same-company Client Admin resolves the request by request type -> production still shows at least the `BUM_SIGNUP` path as live.
- Claim and represented-contact workflow: Bum opportunity claims persist `opportunity_claims` plus `opportunity_claim_contacts`, and My Contacts mixes claim-backed and extension-captured represented contacts -> current live volume is still thin, so Product Ops keeps this as a watch surface rather than a new active queue item.
- Managing Bum workflow: Admin enables `is_managing_bum` on a Bum profile and assigns the Managing Bum terms record -> the Managing Bum can invite other Bums -> `profile-bootstrap` attaches invited signups to `bum_team_memberships` -> team payouts and manager-allocation rows still have no live production volume.

- Finance workflow: customer payment report -> invoice -> payout -> future Managing Bum allocation, with exception lanes still pending before first live finance volume.
