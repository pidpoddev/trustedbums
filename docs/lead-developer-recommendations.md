## Executive Read

The sharpest queue is now led by two verified production-facing risks and one release-gate gap. Live Supabase inventory on 2026-05-29 still shows `extension-api-v1` deployed with `verify_jwt = false`, while the source-controlled handler is meant to enforce a bearer token and deny Bum `customerTargets`. The public `send-website-email` path is still anonymous, wildcard-CORS, and missing server-side anti-abuse checks. Separately, `/admin/performance` and `performance_metric_events` are live, but the business-access rule for telemetry/admin observability was missing until this run and route-level allow/deny proof is still absent.

The next tier is also clearer with fresh evidence. `pnpm run build` still emits one `1,943.89 kB` JS chunk (`510.23 kB` gzip). `npm audit --omit=dev --json` still flags shipped `react-router-dom@6.30.1` and direct `postcss@8.5.6`, plus a newly surfaced direct advisory path through `@clerk/chrome-extension` that should be triaged separately from the runtime portal patch. Live Supabase edge-function logs now confirm the telemetry path is active but incomplete: `performance-beacon` has both accepted `202` writes and rejected `403` writes in the last 24 hours. External trust verification is still runner-limited on 2026-05-29 because `curl -I -L --max-time 20 "$QA_BASE_URL"` and `curl -I https://trustedbums.com` timed out during DNS resolution from this environment.

## Cross-Backlog Dependencies

- The Customer Targets security work depends on the current business-access rule staying explicit: unrelated Bum target browsing is denied by default until product approves a narrower pre-acceptance field subset.
- Public trust and security are linked on the website contact flow. Anti-abuse work reduces operational spam, sender-reputation damage, and buyer distrust simultaneously.
- `/admin/performance`, `performance_metric_events`, and `admin_dashboard_summary` are now one cross-functional admin-only observability surface. QA, Security, Data, and Product Ops should use the new business-access rule before treating telemetry or admin summary output as release-safe.

- UI and product-trust work on the Bum dashboard and privacy controls should follow the security and trust-boundary queue, but they can be bundled later with the route-splitting pass because both touch first-scan confidence surfaces.
