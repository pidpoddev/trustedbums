### P1 - [TB-0059] Remove the public pre-store extension zip until permissions are narrowed
- Evidence: Live `curl -I -L --max-time 20 https://trustedbums.com/downloads/trustedbums-extension.zip` returned `HTTP/2 200` with `content-type: application/zip` on 2026-06-08, confirming the download is still public. [chrome-extension/trustedbums/manifest.json](/Users/macdaddy/CodexWork/TrustedBums/trustedbums/chrome-extension/trustedbums/manifest.json) still requests `activeTab`, `cookies`, `storage`, and host permissions for LinkedIn, `*.supabase.co`, `api.trustedbums.com`, `trustedbums.com`, and Clerk hosts. Google Search Console’s current Security Issues guidance still treats harmful downloads as part of the site-review path, and Chrome’s current extension guidance still says to request only permissions relevant to the extension’s main function. Sources: [Security issues report](https://support.google.com/webmasters/answer/9044101), [Permission warning guidelines](https://developer.chrome.com/docs/extensions/develop/concepts/permission-warnings).
- Why it matters: A publicly downloadable pre-store extension with broad permissions creates reviewer distrust, increases download-reputation risk on the main domain, and weakens the site’s high-trust B2B posture even if the file is not prominently linked.
- Recommendation: Pull the zip off the public site until the distribution plan is final, then reduce permissions to the narrowest production set and lock extension API CORS to the final extension origin.
- Acceptance criteria: The public site returns a controlled non-public result for `/downloads/trustedbums-extension.zip` unless distribution is intentionally re-enabled; extension permissions are reduced to the smallest shipped set; and extension API CORS is restricted to the final production extension origin.
