## Current Standards And Time-Sensitive Notes

- Google Search Console says indexing triage should check Manual Actions and Security Issues before assuming a crawl-only problem. Sources: [Manual actions report](https://support.google.com/webmasters/answer/9044175?hl=en), [URL Inspection tool](https://support.google.com/webmasters/answer/9012289?hl=en).
- Google Search Central still says sitemaps can be surfaced through Search Console, the Search Console API, or a `Sitemap:` line in `robots.txt`. Source: [Build and submit a sitemap](https://developers.google.com/search/docs/crawling-indexing/sitemaps/build-sitemap).
- Google still treats crawlable internal links as discovery signals and treats manipulative link schemes as spam-policy risk. Sources: [Link best practices](https://developers.google.com/search/docs/crawling-indexing/links-crawlable), [Google Search Essentials](https://developers.google.com/search/docs/essentials).
- Bing still recommends IndexNow for faster cross-engine discovery and still supports direct URL submission through Webmaster Tools. Sources: [IndexNow](https://www.bing.com/indexnow), [URL Submission](https://www.bing.com/webmasters/help/url-submission-62f2860b).
- Gmail sender guidance still expects DMARC on the sending domain and alignment through SPF or DKIM on the visible `From:` domain. Source: [Email sender guidelines](https://support.google.com/mail/answer/81126?hl=en).
- Microsoft’s current SmartScreen guidance still ties warnings to site reputation, content, downloads, TLS posture, and user feedback rather than to a single exposed owner-side signal. Sources: [Microsoft Defender SmartScreen overview](https://learn.microsoft.com/en-us/windows/security/operating-system-security/virus-and-threat-protection/microsoft-defender-smartscreen/), [Unexpected SmartScreen block or warning on website](https://learn.microsoft.com/en-us/troubleshoot/microsoft-edge/development/unexpected-block-warning).
- OWASP Secure Headers still treats HSTS, CSP, framing protection, referrer policy, and permissions policy as active browser trust controls worth keeping in deployed checks. Source: [OWASP Secure Headers Project](https://owasp.org/www-project-secure-headers/index.html).
- Supabase’s current changelog did not surface a new June 2026 trust-specific breaking change for the current contact, tracking, or mailbox edge functions. The most relevant trust-adjacent note remains the documented anti-abuse pressure on Supabase email infrastructure and the standing rule that newly created tables are not auto-exposed to the Data API by default. Source: [Supabase changelog](https://supabase.com/changelog).
