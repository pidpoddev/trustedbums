## Current Standards And Time-Sensitive Notes

- Google Search guidance still says indexing triage should check Manual Actions and Security Issues before assuming a crawl-only problem. Sources: [Manual actions report](https://support.google.com/webmasters/answer/9044175?hl=en), [URL Inspection tool](https://support.google.com/webmasters/answer/9012289?hl=en).
- Google Search Central still says sitemaps can be surfaced through Search Console, the Search Console API, or a `Sitemap:` line in `robots.txt`, and Google still treats crawlable internal links as discovery signals while treating manipulative link schemes as spam-policy risk. Sources: [Build and submit a sitemap](https://developers.google.com/search/docs/crawling-indexing/sitemaps/build-sitemap), [Google Search Essentials](https://developers.google.com/search/docs/essentials).
- Gmail sender guidance still expects SPF, DKIM, and DMARC on the sending domain to improve inbox placement and reduce spam or rejection risk. Source: [Email sender guidelines](https://support.google.com/mail/answer/81126?hl=en).
- Bing still recommends IndexNow, direct URL submission, crawlable internal links, and healthy sitemap submission as supported discovery paths. Sources: [URL Submission](https://www.bing.com/webmasters/help/URL-Submission-62f2860b), [Bing Webmaster Guidelines](https://www.bing.com/webmasters/help/webmaster-guidelines-30fba23a).
- Microsoft's current SmartScreen guidance, updated last month, still ties warnings to phishing or malware signals, download reputation, TLS posture, and user feedback. Sources: [Microsoft Defender SmartScreen overview](https://learn.microsoft.com/en-us/windows/security/operating-system-security/virus-and-threat-protection/microsoft-defender-smartscreen/), [Microsoft Edge support for Microsoft Defender SmartScreen](https://learn.microsoft.com/en-us/deployedge/microsoft-edge-security-smartscreen).
- Supabase's current docs, updated last week, still say Edge Function `verify_jwt` is enabled by default, API keys belong in the `apikey` header rather than the `Authorization` header, and teams can deliberately turn the platform check off when they enforce their own auth mode in code. Sources: [Function Configuration](https://supabase.com/docs/guides/functions/function-configuration), [Securing Edge Functions](https://supabase.com/docs/guides/functions/auth), [Authorization headers](https://supabase.com/docs/guides/functions/auth-headers).
- Supabase's current changelog still carries the March 2026 hosted rate limit on recursive or nested Edge Function calls and did not surface a new June 2026 trust-specific breaking change for the current contact, tracking, mailbox, or claim-reply functions. Source: [Supabase changelog](https://supabase.com/changelog).
- OWASP Secure Headers still treats HSTS, CSP, framing protection, referrer policy, and permissions policy as active browser-trust controls worth keeping in deployed checks. Source: [OWASP Secure Headers Project](https://owasp.org/www-project-secure-headers/).
